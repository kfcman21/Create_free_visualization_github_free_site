소개

비용 걱정 없이 나만의 웹사이트를 만들고, 전문적인 시각화 자료를 제작하고 싶은가? 이 가이드에서는 GitHub Pages와 제미나이(Gemini) '젬 지침'을 활용하여 무료로 웹사이트를 만들고 콘텐츠를 시각화하는 방법을 쉽고 자세하게 설명함.

무료 웹사이트 만들기: GitHub Pages

GitHub Pages는 깃허브에서 무료로 제공하는 웹 호스팅 서비스임. 여러분이 만든 웹사이트 파일을 인터넷에 올려 다른 사람들이 볼 수 있도록 하며, 개인 블로그, 포트폴리오, 간단한 프로젝트 홈페이지 등을 만들 때 매우 유용함. 코딩을 잘 몰라도 시작할 수 있음.

💡 누가 GitHub Pages를 사용할 수 있나?

  • GitHub Free 계정을 가진 조직의 공개 리포지토리 및 GitHub Pro, GitHub Team, GitHub Enterprise Cloud, GitHub Enterprise Server 계정을 가진 조직의 공개 및 비공개 리포지토리에서 사용 가능함.

준비물

  • 깃허브(GitHub) 계정
  • 아주 간단한 HTML 파일 (초보자도 쉽게 만들 수 있음)

만드는 4단계

  1. 새로운 저장소(Repository) 만들기:
    • 깃허브에 로그인한 후, 새로운 저장소를 만듦.
    • 저장소 이름은 여러분의 깃허브 사용자 이름 + .github.io 형식으로 지어주어야 함 (예: yourusername.github.io). 이렇게 해야 사용자 페이지가 됨.
    • Public으로 설정하고 README.md 파일도 추가하여 저장소를 만듦.
  2. HTML 파일 업로드하기:
    • 방금 만든 저장소로 들어가서 Add file > Upload files를 클릭함.
    • index.html이라는 이름의 파일을 만들거나 업로드해야 함. 이 파일이 웹사이트의 첫 화면이 됨.
    • 예시 HTML 코드를 복사하여 파일에 넣고 저장(커밋)함.
  3. GitHub Pages 활성화하기:
    • 저장소 메인 화면에서 Settings (설정) 탭으로 이동함.
    • 왼쪽 메뉴에서 Pages를 클릭함.
    • "Build and deployment" 섹션의 "Source"에서 Deploy from a branch (브랜치에서 배포)를 선택함.
    • "Branch"에서 main 브랜치를 선택하고 Save (저장) 버튼을 누름.
  4. 결과 확인하기:
    • 몇 초에서 몇 분 후 (최대 10분 정도 걸릴 수 있음), Pages 설정 화면에 여러분의 웹사이트 주소 (URL)가 나타남.
    • 이 URL로 접속하면 여러분만의 무료 웹사이트를 확인할 수 있음.

팁

  • index.html 파일을 수정하고 저장하면 몇 초에서 몇 분 안에 웹사이트에 바로 반영됨.
  • ChatGPT나 Claude 같은 AI 챗봇에게 HTML 코드를 짜달라고 요청하면 더 멋진 디자인을 쉽게 만들 수 있음.
  • Jekyll 테마 사용: GitHub Pages는 Jekyll이라는 정적 사이트 생성기를 지원하며, 미리 만들어진 테마를 로드하여 디자인을 향상시킬 수 있음. _config.yml 파일을 편집하여 사이트 제목 및 설명을 변경할 수 있음.
  • 커스텀 도메인 연결: github.io 기본 도메인 대신 직접 구매한 커스텀 도메인을 연결하여 브랜딩 및 SEO에 유리하게 사용할 수 있음. 리포지토리 Settings > Pages에서 커스텀 도메인을 입력하고, 도메인 등록 업체의 DNS 설정(CNAME, A 레코드 등)을 변경해야 함. 루트 도메인 연결 시 www 서브 도메인도 함께 연결하는 것이 권장됨.
  • 사용자 지정 404 페이지 만들기: 사용자가 존재하지 않는 페이지에 액세스할 때 사용자 지정 404 오류 페이지를 표시할 수 있음. 사이트의 게시 원본 루트에 404.html 또는 404.md 파일을 추가하면 됨.
  • HTTPS 적용하여 사이트 보호: HTTPS는 사이트 트래픽을 암호화하여 스누핑이나 변조를 방지함. 리포지토리 Settings > Pages에서 'HTTPS 적용'을 선택할 수 있음. 혼합 콘텐츠 문제가 발생하면 HTML에서 http://를 https://로 변경하여 모든 자산이 HTTPS를 통해 제공되도록 해야 함.
  • CI/CD (지속적 통합/지속적 배포) 자동화: GitHub Actions는 빌드, 테스트, 배포 파이프라인을 자동화할 수 있는 CI/CD 플랫폼임. 저장소 내의 이벤트에 의해 워크플로우가 트리거되며, .github/workflows 디렉토리에 YAML 파일로 정의됨.
  • 검색 엔진 최적화 (SEO): Google 검색은 크롤링, 색인 생성, 검색결과 게재의 세 단계로 작동함. 사이트맵, RSS, robots.txt 등의 요소를 활용하여 SEO를 개선할 수 있음. jekyll-seo-tag 및 jekyll-sitemap 플러그인도 유용함.

전문적인 시각화 자료 만들기: 제미나이(Gemini) '젬 지침'

요즘 AI 도구들이 문서를 시각화하는 능력이 정말 좋아졌지만, 대부분 '크레딧'이라는 비용을 사용해야 하는 단점이 있음. 하지만 제미나이의 '젬 지침'을 활용하면 크레딧 소모 없이 전문적인 시각화 자료를 무료로 만들 수 있음.

왜 제미나이 젬 지침이 좋을까?

  • 무료: 젠스파크(ZenSpark) 같은 다른 좋은 도구들은 비용이 발생하지만, 제미나이는 무료로 이런 고급 기능을 사용할 수 있음.
  • 방대한 데이터 처리: 제미나이는 무료 플랜에서도 많은 HTML 소스(웹사이트를 만드는 코드)를 처리할 수 있는 뛰어난 컨텍스트 용량을 지원함. 콘텐츠 시각화의 기본 원리는 시각화된 요소를 HTML로 구현하는 것이기 때문에 이는 매우 중요함.
  • 최신 AI 모델 활용: 항상 최신 AI 모델을 사용할 수 있어서 더 나은 결과물을 기대할 수 있음.
  • 쉬운 접근성: 예전에는 디자이너나 데이터 분석 전문가만 만들 수 있었던 전문적인 시각화 자료를 AI의 도움으로 누구나 쉽게 만들 수 있게 해줌. 복잡한 소프트웨어를 배울 필요 없이 자연어로 명령만 하면 됨.
  • 원본 지침 제공: 사용자가 직접 지침을 수정하고 커스터마이징할 수 있도록 완전한 원본 지침(GMG 지침)을 제공함. 이는 AI의 확률적 추론 및 오해석으로 인한 한계를 극복하고 최적의 결과를 얻기 위함임.

무엇을 만들 수 있나?

  • 인터랙티브 대시보드: 정보를 한눈에 볼 수 있는 웹 페이지 형태의 자료.
  • 비주얼 인포그래픽 보고서: 그림과 핵심 내용을 강조한 시각적인 보고서.
  • 원페이지 요약 보고서: 핵심 내용만 간결하게 정리한 한 페이지짜리 보고서.

두 가지 젬 지침 유형: '고정형' vs '자유형'

두 지침은 사용자 콘텐츠의 웹 시각화라는 동일한 목표를 가지고 있지만, 철학과 작동 방식, 결과물의 특성에서 근본적인 차이를 보임.

고정형 (Fixed Type) 지침

  • 역할: 인테리어 디자이너나 출판 편집자와 같음.
  • 템플릿 기반: 이미 검증된 견본, 즉 템플릿을 먼저 선택하고 콘텐츠를 그 견본에 정해진 틀에 맞게 재구성하고 채워 넣는 방식으로 작동함 (총 3개의 템플릿 포함).
  • 콘텐츠 처리: 소스에서 필요한 텍스트와 데이터를 추출하여 템플릿에 고정된 공간에 맞추기 위해 텍스트를 공격적으로 요약하고 재가공함.
  • 결과물 특징:
    • 정형화된 대시보드 레이아웃을 채택함.
    • 검증된 템플릿 기반이므로 디자인적 완성도나 레이아웃 안정성이 뛰어남.
    • 결과물의 품질과 형태가 항상 보장되므로 안정적임.
    • 다만, 고정된 틀에 콘텐츠를 맞추는 과정에서 일부 정보가 요약되거나 누락될 수 있음.

자유형 (Free Type) 지침

  • 역할: 건축가나 엔지니어와 같음.
  • 템플릿 없음: 템플릿이 없음.
  • 콘텐츠 처리: 의미 단위로 분석하여 모든 서술형 문장을 보고서 스타일의 개조식으로 변환함.
  • 결과물 특징:
    • 매번 다른 형식의 대시보드가 생성될 수 있음.
    • 콘텐츠의 양과 구조에 따라 컴포넌트의 수와 배치가 달라져서 결과물의 최종 형태가 가변적임.
    • 가장 논리적인 형태로 표현되며 콘텐츠의 양이나 복잡도에 따라 자연스럽게 확장되거나 축소되는 특징이 있음.

어떻게 사용하나?

  • 지침 다운로드: 소스에 언급된 블로그 주소(영상 설명란과 고정 댓글 참고)에서 '젬 지침'을 다운로드하고 압축을 풂.
  • 지침 설치:
    • 고정형: 젬 이름을 '고정형' 등으로 입력한 후, 다운받은 '콘텐츠고정형' 지침을 붙여 넣고, 함께 있는 3가지 템플릿도 추가해서 설치함.
    • 자유형: 이름을 입력하고 '콘텐츠 자유형' 지침을 열어 내용을 붙여 넣기 함. 이때, 젬 자체적인 문제로 이전 템플릿이 포함되어 있다면 반드시 삭제해야 함.
  • 실행: 제미나이에서 해당 젬을 실행하고 '시작' 또는 '고'를 입력한 다음, 원하는 시각화 종류의 번호를 선택하거나 시각화할 텍스트, 문서, URL 등을 입력하면 됨.

활용 팁 및 한계

  • 제미나이 젬 지침이 HTML 코드를 포함한 시각화 자료를 만들어주므로, 이 결과물을 GitHub Pages에 업로드하여 여러분의 무료 웹사이트에 전문적인 시각화 콘텐츠를 추가할 수 있음.
  • 젬 지침이 매우 정교하게 설계되었음에도 불구하고, 인간 언어와 맥락의 본질적 모호성 및 각 젬 페르소나의 구조적 한계로 인해 매번 완벽한 구조의 시각화 자료가 나오는 것은 불가능함.
  • 원페이지 요약 보고서의 경우, 소스 내용의 분량에 따라 콘텐츠가 잘리는 문제가 발생할 수 있음. 현재는 젬 자체적인 문제이며, 이 경우 직접 코드를 편집하여 조정해야 함.
  • 궁금한 점이 있다면 오픈채팅방을 통해 가장 빠르게 지원받을 수 있음.

결론

GitHub Pages는 여러분의 웹사이트를 무료로 호스팅할 수 있게 해주고, 제미나이 젬 지침은 여러분의 콘텐츠를 무료로 전문적인 시각화 자료로 만들어주는 도구임. 이 두 가지를 함께 활용하면 비용 걱정 없이 여러분만의 멋진 웹사이트와 전문적인 시각화 콘텐츠를 만들 수 있을 것임.
